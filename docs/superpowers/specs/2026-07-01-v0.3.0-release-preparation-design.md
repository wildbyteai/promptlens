# PromptLens v0.3.0 Release 准备设计

日期：2026-07-01

## 背景

PromptLens 当前 `manifest.json` 版本为 `0.3.0`，仓库尚无 git tag 和 GitHub Release。项目已经落地 v0.3.0 的主要功能，包括自定义模板、Provider presets、截图框选快捷键、结果导出、模型测试、token usage、可选本地历史记录、基础 i18n 和 Chrome Web Store 发布准备。

本次目标是把 v0.3.0 作为首个正式 GitHub Release 发布，并补齐后续 Chrome Web Store 提交所需的打包与文档材料。

## 目标

1. 新增可重复执行的 Chrome 扩展打包脚本。
2. 生成并验证 `dist/promptlens-v0.3.0.zip`。
3. 新增 Chrome Web Store 发布材料文档。
4. 创建正式 GitHub Release `v0.3.0` 并上传 zip artifact。
5. 将官网用户可见下载入口更新到具体 Release 页面。

## 非目标

- 不提交 Chrome Web Store 审核。
- 不修改扩展运行逻辑或权限模型。
- 不引入 npm、构建系统或第三方依赖。
- 不重写已有 README 或官网视觉设计。
- 不把无关未跟踪文件纳入 Release 准备提交。

## 打包脚本设计

新增 `scripts/package-extension.sh`，从仓库根目录运行。脚本读取 `manifest.json` 中的版本号，默认输出：

```text
dist/promptlens-v0.3.0.zip
```

脚本只负责 Chrome 扩展运行包，不负责创建 tag、GitHub Release 或商店提交。

### 包含文件

扩展包应包含运行所需文件：

- `manifest.json`
- `background.js`
- `content.js`
- `history-store.js`
- `options.html`
- `options.css`
- `options.js`
- `result.html`
- `result.css`
- `result.js`
- `templates.js`
- `_locales/**`
- `icons/**`

### 排除文件

扩展包不应包含仓库维护、文档、官网或临时产物：

- `.git/**`
- `docs/**`
- `site/**`
- `dist/**`
- `README*.md`
- `SECURITY.md`
- `CONTRIBUTING.md`
- `LICENSE`
- `skills-lock.json`
- README 截图源文件，例如 `promptlens-*-readme.png`
- 其他非扩展运行文件

## zip 验证设计

脚本生成 zip 后进行基础内容验证。验证失败时退出非零状态，避免上传错误 artifact。

必须验证：

1. zip 文件存在且非空。
2. zip 内包含 `manifest.json`。
3. zip 内包含 `_locales/` 资源。
4. zip 内包含 `icons/` 资源。
5. zip 内包含主要 HTML、CSS、JS 文件。
6. zip 内不包含 `docs/`、`site/`、`.git/`、`dist/`、README 截图源文件或 `skills-lock.json`。

验证输出应清楚说明 zip 路径和通过状态，便于复制到 Release 流程中。

## Chrome Web Store 文档设计

保留现有 `docs/chrome-web-store/README.md` 作为总览文档，并新增三份更细材料：

### `docs/chrome-web-store/store-listing.md`

用于准备商店详情页内容，包含：

- 扩展名称。
- 简短描述。
- 详细描述。
- 核心功能点。
- 权限说明。
- 截图建议。
- 暂不承诺内容。

文案应和当前 README、manifest 权限、隐私边界保持一致。

### `docs/chrome-web-store/privacy-practices.md`

用于辅助填写 Chrome Web Store 隐私实践表单，包含：

- 项目无后端、无遥测的边界。
- API Key 存储在 `chrome.storage.local`。
- 图片只发送到用户配置的 AI Base URL。
- 历史记录默认关闭；开启后只保存文本结果、来源域名和模板元数据，不保存图片缩略图。
- 远程图片读取权限为可选授权。
- 第三方模型服务的数据处理责任归属于用户选择的服务商。

### `docs/chrome-web-store/release-checklist.md`

用于发布前最终核查，包含：

- manifest 版本和权限检查。
- 打包脚本执行。
- zip 内容验证。
- 本地加载解压扩展。
- 右键图片分析。
- 框选截图分析。
- 快捷键触发。
- 设置页保存。
- 模型测试与 token usage。
- 自定义模板导入导出。
- 本地历史记录默认关闭和清空能力。
- GitHub Release artifact 检查。
- 官网链接检查。

## GitHub Release 设计

正式发布 `v0.3.0`：

1. 创建 annotated tag `v0.3.0`。
2. 创建 GitHub Release，标题为 `v0.3.0`。
3. 上传 `dist/promptlens-v0.3.0.zip`。
4. Release notes 聚焦 v0.3.0 用户可见能力和安装说明。

Release notes 应包含：

- 自定义输出模板。
- Provider presets。
- `Alt+Shift+S` 截图框选快捷键。
- 结果导出：JSON 和 Markdown。
- 模型测试与 token usage 显示。
- 可选本地历史记录。
- 基础 i18n。
- Chrome Web Store 发布准备材料。

## 数据流与职责边界

发布准备流程的数据流如下：

1. `manifest.json` 提供版本号。
2. `scripts/package-extension.sh` 根据允许清单收集扩展运行文件。
3. 脚本输出 `dist/promptlens-v0.3.0.zip`。
4. 验证步骤检查 zip 内容边界。
5. Git tag 和 GitHub Release 使用同一个版本号 `v0.3.0`。
6. 官网链接指向对应 GitHub Release 页面。

各部分职责保持分离：

- 打包脚本只负责 artifact。
- Chrome Web Store 文档只负责审核材料。
- GitHub Release 只负责公开版本发布。
- 官网只负责下载入口指向。

## 错误处理

- 如果无法从 `manifest.json` 读取版本号，打包脚本应失败并说明原因。
- 如果必需文件不存在，打包脚本应失败并列出缺失文件。
- 如果 zip 验证发现禁入文件，脚本应失败并列出匹配项。
- 如果 `v0.3.0` tag 已存在，应停止并确认，不覆盖公开 tag。
- 如果 GitHub Release 已存在，应优先检查内容，而不是盲目重建。
- 如果 `gh` 未登录或没有权限，应停止并提示用户处理认证。

## 验证计划

实施完成后至少运行：

1. `bash scripts/package-extension.sh`
2. `unzip -l dist/promptlens-v0.3.0.zip`
3. `git status --short`
4. `gh release view v0.3.0`（Release 创建后）

必要时还应通过 Chrome 加载解压后的扩展做手动验证，但本次自动化验收以包内容和发布资产一致性为主。

## 验收标准

1. `scripts/package-extension.sh` 可重复生成 `dist/promptlens-v0.3.0.zip`。
2. zip 包只包含扩展运行文件，不包含文档、官网、git 元数据或本地杂项。
3. 三份 Chrome Web Store 文档存在且与当前实现一致。
4. GitHub 上存在 `v0.3.0` Release，且附带 zip artifact。
5. 官网下载入口指向 `https://github.com/wildbyteai/promptlens/releases/tag/v0.3.0`。
6. 工作区不会误纳入与本次发布准备无关的未跟踪文件。
